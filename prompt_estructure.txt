Project context:
This is Practice 4: Implementation of a National Electoral Count System for Bolivia.

The system has two decoupled pipelines:

1. RRV - Recuento Rápido de Votos
- Input: PDF/image of electoral acts, mobile app photo, or SMS.
- Flow: input -> OCR or SMS parser -> RRV validator -> functional logs/events -> MongoDB cluster -> dashboard.
- Purpose: preliminary results, low latency, eventual consistency.
- Cluster 1: MongoDB Replica Set.
- Main collections:
  - rrv_actas
  - rrv_eventos
  - rrv_logs
  - rrv_metricas_tecnicas

2. CO - Cómputo Oficial
- Input: official CSV/transcription data.
- Flow: CSV -> official automation -> official validator -> PostgreSQL cluster -> dashboard.
- No OCR, no images, no SMS in this flow.
- Purpose: official auditable results, strong consistency.
- Cluster 2: PostgreSQL replicated.
- Main tables:
  - territorios
  - recintos
  - mesas
  - usuarios_transcripcion
  - actas_oficiales
  - auditoria_oficial

Dashboard:
The dashboard compares RRV vs official count and must later show:
- Participation
- Total votes: valid, blank, null
- Votes by party
- Act status: received, processed, pending, observed, rejected, duplicated, published
- Participation rate
- Percentage by party
- Victory margin
- Processing speed
- Differences between quick count and official count
- Geographic analysis
- Transparency and access to digital acts
- Technical indicators: latency, throughput, availability, security
- Anomalies and logs
